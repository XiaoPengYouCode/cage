# FJW Golden Cases

The repository now has a small golden fixture path for regression checks, but it still does not contain historical MATLAB + Abaqus runtime outputs.

Missing historical outputs:

- `Force_*.mat`
- `U1_ele_nod_dir*.mat`
- `obj_bo*.mat`
- `ob*.mat`
- full Abaqus `.odb` files from `Three_Force_Time_Dep.m`

Golden cases:

- `datasets/fjw_golden/minimal_mma/`: a compact MMA one-step fixture generated by `scripts/build_fjw_golden_case.py`.
- Reference-scale dry-run checks should compare generated `.inp` structure, node/element sets, material buckets, and load blocks. They are input-equivalence checks only.

Boundary:

Without historical runtime outputs, validation reports must say that MATLAB + Abaqus numerical equivalence is not proven. Python-generated Abaqus results can become a new golden source after a real Abaqus run is captured with manifests and checksums.
